# Security policy

Не публикуйте API-ключи, токены, пароли, приватные ключи или реальные персональные данные в issue и pull request.

При обнаружении секрета:

1. немедленно отзовите его у провайдера;
2. выпустите новый секрет;
3. удалите значение из текущего дерева и истории Git;
4. храните замену в GitHub Actions Secrets или переменных окружения.
