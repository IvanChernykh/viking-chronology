# Архитектура

## Потоки исполнения

1. **React UI** управляет фильтрами, карточками, аудио и хронологией.
2. **React Three Fiber** изолирует WebGL-сцену и загружается лениво отдельным чанком.
3. **Three.js** рендерит глобус, маршруты, маркеры и процедурные корабли.
4. **Web Audio API** создаёт звуковую среду без внешних аудиофайлов.
5. **Vite** выпускает две сборки: оптимизированную web-версию и автономный IIFE HTML.

## Границы отказа

- 3D-сцена обёрнута в React Error Boundary.
- Потеря WebGL-контекста переводит приложение в контролируемый fallback с перезапуском.
- При отсутствии WebGL 2 основной интерфейс не падает.
- Звук инициализируется только после пользовательского действия.

## Производительность

- Three.js загружается после первичного интерфейса через `React.lazy`.
- Хронология фиксирует React-состояние не чаще 20 FPS на desktop и 11 FPS на mobile.
- Движение кораблей остаётся независимым и выполняется внутри render loop.
- Автопрофиль качества может только один раз понизить нагрузку после измерения FPS; колебаний DPR нет.
